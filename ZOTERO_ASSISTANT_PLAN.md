# Zotero 助手 v1 插件计划

## Summary

构建一个 Zotero 9 桌面端插件内置 Agent，不做 MCP，不开本地 HTTP 接口，不支持 Claude、Cursor、ChatGPT 等外部客户端直接控制。第一版核心体验是：用户在 Zotero 右侧侧边栏明确指定一个任务后，AI 以任务线程 harness 方式多轮推进，读取必要上下文、申请按钮授权、调用 Zotero 和已授权插件命令、执行操作、记录日志，并支持可逆撤销。

助手没有默认任务，不默认进入“整理书架”。用户必须先指定任务，例如整理某个 collection、修改某项设置、调用某个已授权插件命令、读取选中文献并生成笔记等。如果用户任务描述过于模糊，AI 必须先追问关键目标，而不是擅自执行。

默认安全模式为 AI 审核模式：低风险操作自动执行，高风险操作通过侧边栏按钮授权。另提供确认模式和完全开放模式。

## Key Changes

- 插件形态：Zotero 9 插件，TypeScript 工程化实现，交付源码开发版和 unsigned XPI。
- UI：使用 Zotero 原生控件风格实现右侧侧边栏。
- 模型：OpenAI 兼容接口，用户配置 base URL、API key、模型名；API key 保存到 Zotero preferences，并提示本机存储风险。
- 工具调用：function calling 优先；模型不支持时暂停任务并提示更换模型，不做 JSON 降级。
- 任务模型：一次只允许一个活动任务，内部使用 harness 状态机，UI 显示轻量状态、子任务摘要、授权卡片、日志和结果。
- 日志：结构化事件日志保存在本地插件数据目录，默认保留 30 天。
- 恢复：任务可继续、取消或回滚已执行的可逆操作。
- 模型失败：自动重试 3 次，仍失败后暂停并显示原因。

## Implementation

- 建立 Zotero 9 插件工程，包括 TypeScript 源码、构建脚本、XPI 打包脚本、插件入口、侧边栏、设置页、任务 harness 和 Zotero 工具层。
- 实现三种安全模式：确认模式、AI 审核模式、完全开放模式。
- 实现任务线程 harness：
  - 用户必须先输入明确任务。
  - AI 可以在一个任务内多轮规划、读取上下文、申请授权、执行工具、观察结果并继续推进。
  - 同一时间只允许一个活动任务。
  - UI 只展示轻量任务状态，不暴露完整内部状态机。
- 低风险自动操作包括创建少量 collections、添加标签、创建或追加笔记、将条目加入 collection。
- 默认不从原 collection 移除条目。
- 单次任务最多新建 5 个 collections，最多处理 100 条文献。
- 高风险操作包括删除、设置修改、批量归类、元数据覆盖、插件命令调用、全文读取和大范围上下文读取。
- 删除只允许移到 Zotero 回收站，不允许永久删除。
- 授权通过侧边栏按钮完成，提供“允许本次”“记住此命令”“拒绝”。
- 授权卡片默认显示 AI 的自然语言摘要，要求 AI 说明自己请求权限是为了做什么。
- 高风险操作提供“查看详情”展开项。
- 其他插件调用采用菜单命令桥接，按具体命令授权，不调用未知插件内部 API。
- Zotero 工具层覆盖：
  - 搜索条目。
  - 读取当前 selection。
  - 读取当前 collection。
  - 读取可见列表摘要。
  - 读取元数据、摘要、标签、collections 和笔记摘要。
  - 按任务授权读取全文、批注和更大范围文库上下文。
  - 创建 collections。
  - 将条目加入 collections。
  - 添加标签。
  - 创建或更新笔记。
  - 更新元数据。
  - 修改可见 Zotero 设置。
  - 移到回收站。
  - 记录可逆操作前后状态并支持最近操作撤销。
- 设置修改范围以 Zotero 可见设置为主；隐藏偏好只读或需要完全开放模式加强确认。
- 其他插件权限范围：
  - 支持 Zotero 加授权插件。
  - 优先通过菜单命令桥接。
  - 用户按具体命令授权。
  - 授权入口在任务中的按钮卡片上，而不是要求用户先去配置页预设权限。
- 数据发送策略：
  - 调用 OpenAI 兼容接口时，可以发送用户按任务授权范围内的数据。
  - 默认上下文为当前 selection、当前 collection 和可见列表摘要。
  - AI 需要更多上下文时必须说明原因并通过按钮申请。
- 任务执行策略：
  - 高影响不确定才追问用户。
  - 小的不确定点按 AI 推荐默认继续。
  - 用户任务描述模糊时，AI 先追问目标。
  - 例如用户只说“帮我整理这些文献”但没有说明整理标准时，AI 不能直接修改书架，必须先问整理目标。

## Test Plan

- 使用测试 Zotero profile 和样本文库做真实冒烟测试。
- 验证插件安装、启动、侧边栏、设置页和模型配置。
- 验证 function calling 成功路径、失败重试 3 次、失败暂停。
- 验证用户未指定任务时，助手不会默认执行“整理书架”或其他操作。
- 验证模糊任务会先追问关键目标。
- 验证最多创建 5 个 collections、最多处理 100 条文献。
- 验证自动归类只添加 collection membership，不移除旧归属。
- 验证上下文读取、高风险操作和插件命令都需要正确授权。
- 验证“允许本次”“记住此命令”“拒绝”三种按钮行为。
- 验证可逆操作撤销。
- 验证结构化日志保存、30 天保留和清除。
- 验证至少一个用户本机已安装插件的菜单命令桥接。

## Assumptions

- 第一版以 Zotero 9 为主，manifest 兼容范围暂设为 Zotero 7 到 9，以便安装验证和定位真实 API 差异。
- 当前没有已有源码，实施时从新插件工程开始。
- 使用 Zotero 内部 JavaScript API，不直接写 `zotero.sqlite`。
- API key 存入 Zotero preferences 是可接受的简化方案，但必须提示风险。
- “尽量全开放”不包括永久删除、任意隐藏偏好写入或未知插件内部 API 调用。
- “整理书架”只是一个可由用户发起的任务类型，不是助手启动后的默认任务。
